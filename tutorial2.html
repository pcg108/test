Tutorial 2

Key Points

After watching the video, consider the following key points:

  • C Programs always begin with the Preprocessor directives, which give the pre-processor (which makes minor modifications to the program before handing it to the compiler) certain information it needs to understand the rest of the program.
    • Preprocessor directives always begin with the # symbol.
    • In this lesson, we discussed the #include directive, which we generally use to "include" the information in certain libraries into our program:
      • A library is a file that defines certain functions and features that we use in the program. In this case, we used the stdio.h library.
      • To include a library, use #include . To use another library, like the stdlib library, use #include , and so on.
  • C Programs are composed of functions, which are groups of statements.
    • The "main" function is required in every C program. The first thing the compiler looks for after getting the program from the preprocessor is the main function.
    • Functions always have the form:
      • return-type name() { statements }
    • Most functions will return (give back) a certain value to whatever called it. In the case of the main() function, the compiler is the one that calls it, so main() will return a value to the compiler
      • Functions do not necessarily have to return a value, so in that case, the return-type will be void.
      • main() always returns an integer to the compiler.
    • The name of the main() function is, obviously, "main". Functions can be named with any mix of letters, numbers, and underscores (_), but must begin with a letter or underscore. The name is followed by a pair of parenthesis, which we will discuss later.
    • The parenthesis are followed by a pair of curly braces { }, in which the statements go.
  • The statements are the lowest level of our heirarchy. You can think of them as the lines that actually perform tasks in our program.
    • Statements always end with semicolons ( ; ). Think of semicolons as periods at the end of a sentence.
    • We learned how to print things out (like "Hello, World"), with the following statement:
      • printf("Hello World");
      • This displays whatever you put inside of the quotation marks.
    • We also learned about the return statement. Recall that the main() function returns a value to the compiler. This is the line where we actually return that value.
    • In most cases, we always return the value 0 in main(), which is interpreted by the compiler as a sign that the program compiled properly. We do this with:
      • return 0;
  • Lastly, we learned how to compile and run programs, as well as how we will use Command Prompt.
    • ?Think of Command Prompt, for our purposes, as a kind of text-based file explorer. We navigate to different folders by using the "cd" command, followed by the folder. For example, "cd Desktop".
    • If we want to navigate to a certain folder that is inside of the folder we are already in, we can just use "cd folder-name". Otherwise, we can use "cd entire-path-to-folder"
    • To C programs from command prompt, first make sure that you are in the folder in which your file is located. Then type:
      • gcc program-name.c -o executable-name
      • "gcc" calls the C compiler to compile the "program-name.c file". It compiles is into an object file (machine code file) with the command "-o", and turns that into an executable with the name "executable-name.exe"
      • We can then run the executable by simply typing "executable-name" in Command Prompt (making sure that we are in the folder of the executable)
